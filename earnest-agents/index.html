Earnest Agents

Earnest Agents is foundational agentic infrastructure — a platform on which any AI-driven workflow can be built, deployed, and operated at Earnest. It handles multiple agent runtimes, integrates with Slack, GitHub, and our internal observability stack, and supports both on-demand tasks and recurring scheduled operations. This document describes the architecture and the design principles behind it.

Design Principles

Infrastructure, not a product

Earnest Agents is not an application — it is a platform. New agent workflows are added by writing a configuration and a markdown skill file, with no code changes to the platform itself. The goal is infrastructure that other teams can build on.

One worker per conversation

One worker handles one conversation at a time, in order. Follow-up messages are bundled into the current run rather than queued separately — the agent sees the full context without losing correctness.

Pluggable at every layer

The AI model, execution environment, and tool set are all swappable without changes to the platform. Teams can bring their own model or runtime. The system is not locked to any vendor.

Persistent context, not stateless calls

Conversations persist across sessions. Scheduled tasks reuse the same conversation across runs, letting the agent build understanding over time — detecting trends, not just point-in-time snapshots.

Security as a first-class constraint

Every channel has its own authentication layer. Credentials are scoped per user and never shared across agents. Execution environments are ephemeral. Every action the agent takes is an explicit, named operation — not free-form access.

Full observability built in

Every task run is recorded end-to-end. Cost is tracked and surfaced in real time. The scheduled tasks dashboard gives operators full visibility into what ran, what it produced, and what it cost.

What might be possible

The infrastructure is general-purpose — any capability an agent needs can be expressed as a skill or a tool. These are not hypotheticals; each scenario below builds directly on what the platform already does today.

End-to-end feature delivery

An engineer describes a change in plain language. An agent reads the relevant code, writes the implementation, runs the tests, and opens a pull request with a full description — without anyone touching a terminal.

Proactive incident response

A scheduled agent queries New Relic for error rates and latency anomalies every few minutes. When something looks wrong it posts a structured summary to Slack with a direct link — before an on-call engineer would have noticed.

Automated security scanning

An agent runs dependency audits and SAST tools inside an isolated sandbox on a schedule or triggered by a PR. Findings are summarized in plain language and routed to the right team in Slack.

Deployment intelligence

An agent monitors deployment pipelines, correlates a new deploy with a spike in errors, and either flags it immediately or escalates with the evidence already assembled — minutes, not hours.

Research and synthesis

An agent searches documentation, browses external resources, reads code history, and produces a written analysis or decision brief. New Relic dashboards, GitHub history, Confluence pages — all fair game.

Team-specific agents

Any team can define an agent with a custom identity, its own skills, and access to exactly the tools it needs — deployed without a platform code change. A lending agent, a compliance agent, a data agent — each scoped to its domain.

Continuous code review

Every pull request reviewed automatically against Earnest conventions before a human looks at it. Not a generic linter — a reviewer with context about the codebase that leaves structured comments directly in the PR.

Compounding institutional memory

Agents reuse the same conversation across scheduled runs, accumulating context over time. A monitoring agent that remembers what normal looked like last quarter. An engineering agent that knows how you prefer your code structured.

Arbitrary code execution

Any task that can be expressed as code can be delegated — data transformations, report generation, database queries, API calls. The sandbox is isolated and ephemeral, so there is no risk to production systems.
